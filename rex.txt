""" agent.py -- Multi-tenant LiveKit voice agent. 

Per-call flow: 1. Room arrives -> extract phone from room name OR outbound metadata 2. asyncpg pool query -> phone -> tenant -> location + agent + menu (~5ms, once) 3. Build SYSTEM_PROMPT dynamically for THIS restaurant 4. Function tools are closures over per-call order_tracker + menu_data -> zero I/O on the hot path, all in-process 5. Single pipeline: nova-2 STT + Azure OpenAI + eleven_flash_v2_5 TTS 6. On finalize: DB write + SMS happen in asyncio.gather (non-blocking) """

Endpoints: GET /health — container health check POST /token — generate LiveKit room token for WebRTC calls POST /outbound-call — trigger outbound PSTN call via LiveKit SIP POST /onboard-phone — provision LiveKit + Twilio SIP (idempotent) POST /cleanup-phone — teardown LiveKit + Twilio SIP for a number 

All order/reservation/menu CRUD is handled through conversation-engine and identity-service APIs. This server only does what the LiveKit telephony layer needs. """
